Decode QR codes and 1D/2D barcodes from live camera streams and persist detections as searchable bookmarks.

How it works

For each enabled channel, a helper process runs inside the Event Server. It opens a JPEGLiveSource on the selected camera, runs every frame through the ZXing decoder, and (if enabled) creates a bookmark on the camera timeline for each detection. A watchdog reconnects the live stream automatically if the camera goes offline or the feed stalls.

Configuration

Under MIP Plug-ins > Barcode Reader > Barcode Channels, right-click and choose Create New. Each channel has:

  • Camera - which camera to decode from.
  • Barcode formats - restrict to the code types you actually expect; fewer formats = faster decoding.
  • Decoder options (Try Harder, Auto Rotate, Try Inverted) - accuracy/CPU trade-offs.
  • Create bookmarks for detections - one bookmark per decoded barcode (see below).
  • Target frame rate - hard cap on decode attempts per second.
  • Downscale to width - resize large frames before decoding. Big speed-up on 4K/8K.
  • Duplicate debounce (ms) - suppress repeat detections of the same text.

Saving detections: Bookmarks

When Create bookmarks for detections is on, every decoded barcode creates a Milestone bookmark on the camera's timeline:

  • Header: decoded text (truncated to ~60 chars).
  • Description: Format: {FORMAT} / Channel: {channel name}.
  • Window: 2 s before the detection, 2 s after.

Find them in Smart Client:

  • Search workspace → add the Bookmarks filter → type into the keyword box (matches both Header and Description).
  • Per-camera timeline → bookmark markers are clickable to jump to the frame.

The same Duplicate debounce that governs logging also governs bookmarks - a barcode held in front of the camera only produces one bookmark per debounce window.

Live status panel

While editing a channel, the bottom panel shows:

  • Cam FPS - frames arriving from the camera per second.
  • Decode FPS - frames run through the decoder per second.
  • Inference avg / p95 - time spent in ZXing per frame.
  • Max FPS - theoretical ceiling (1000 / inference_ms_avg).
  • Hint line - green if there's headroom to raise Target FPS, amber if you're at the limit.
  • Helper log - last 40 lines including DETECT entries (newest first).

Camera offline / reconnect behaviour

If no frames arrive for ~8 seconds while the channel is supposed to be live, the helper tears down and re-opens the JPEGLiveSource in place. Status transitions (Running → Error:NoFrames → Running) are reflected in the admin UI in real time. If the helper process itself dies, the Event Server background plugin restarts it within 10 seconds.

Troubleshooting

SymptomFix
No detectionsNarrow the format list to what you expect; enable Try Harder; raise Target FPS.
High CPULower Target FPS, turn off Try Harder / Auto Rotate, enable Downscale.
Status stuck on Error:ConnectCamera or Recording Server unreachable. Check recording server health; helper auto-retries.
Helper exe not foundEnsure BarcodeReaderHelper.exe is in the same folder as BarcodeReader.dll.
Bookmarks not appearingVerify Create bookmarks for detections is checked; check the helper log for BookmarkCreate failed.

Logs

Detailed per-channel output is written to the Event Server log:

C:\ProgramData\Milestone\XProtect Event Server\Logs\

Look for DETECT (successful decodes), STATS (once per second), and STATUS (lifecycle transitions).
