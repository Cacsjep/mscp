Auto Exporter

Automate XProtect video exports on a recurring rule and keep them on disk for a bounded amount of time or space — per job.

Setup

  1. Right-click Jobs in the sidebar → Create New… for each export job.
  2. For each job: name, cameras / groups, format, time range, and the storage path + MAX GB / MAX TIME. Use Verify on ES to confirm the Event Server can reach the path.
  3. Open Executions and click Run Now ▼ to test a job, or build a XProtect Rule using the Execute Auto Export Job action.
  4. Open Status for a per-job dashboard of storage health (path reachable from the Event Server? quota %, free disk, oldest run, etc.).
⚠ Plaintext password storage. Job export passwords are stored as plaintext in the Milestone configuration database. The Management Client encrypts traffic to the server, but anyone with read access to the management DB can extract the passwords.
  • Use export encryption only for non-sensitive exports, or where database access is tightly restricted.
  • If you need stronger protection, gate database access via Milestone role permissions and disable encryption in the job (the export will then be a plain XProtect bundle).

Storage layout (per job)

<Job.StoragePath>\
  28.05.2026_0300\
    <CameraName>\
      ...exported files...
  28.05.2026_0400\
    ...

Each job has its own storage path, MAX GB, and MAX TIME. Two jobs that point at the same path share the same ring buffer (everyone's runs prune together).

Ring buffer

  • Each job's limits apply to its own storage path's run folders only.
  • Cleanup runs before each export and once an hour for every job.
  • Oldest run folders are deleted first until under MAX GB; folders older than MAX TIME are deleted on every pass.
  • 0 = unlimited for either limit.

Execution log

The history shown in the Executions view comes from a single JSONL file at:

%ProgramData%\MSCPlugins\AutoExporter\executions.jsonl

Retention is enforced automatically on every append:

  • Up to 100 entries per job (oldest dropped first within each job)
  • Up to 1000 entries total across all jobs (newest globally kept)

The Executions view shows at most the most recent 500. No manual maintenance.

Time range

Each job specifies Last N [Minutes | Hours | Days | Months]. The range is anchored at the moment the job is triggered, so the same job works for a nightly schedule, a manual Run Now, or an event-driven rule.

Trigger timeRange settingExported window
2026-05-28 03:00Last 2 Days2026-05-26 03:00 → 2026-05-28 03:00
2026-05-28 14:22Last 2 Hours2026-05-28 12:22 → 2026-05-28 14:22

Months = 30 days.

Cameras & camera groups

Each job can target any mix of individual cameras and camera groups. Groups are resolved to their member cameras at the moment of export, so adding/removing cameras from a group flows through automatically.

Rule action

The plugin exposes one action: Execute Auto Export Job. The Rules engine offers individual / folder / ALL targeting automatically. The action fires one of two events on completion:

  • Auto Export: Job Succeeded — chain it to Notification, HTTP Requests, etc.
  • Auto Export: Job Failed — fires on disk-full, missing cameras, “job busy”, etc.

Concurrency

If a rule re-triggers a job while the previous run is still active, the new trigger is skipped, an entry is added to the Executions log, and the Failed event is fired. This prevents disk/IO pile-ups.

Architecture note

Milestone's export APIs (DBExporter / AVIExporter) refuse to run in the Event Server's restricted Service environment. The plugin solves this by spawning AutoExporterHelper.exe next to the plugin DLL — the helper hosts a full standalone SDK environment, logs in to the management server, runs the export, and reports back via stderr + a result JSON. The helper inherits the Event Server service account's credentials, so the same camera-read permissions apply.

Troubleshooting

  • "Storage path not configured on this job." Open the job and set a path that the Event Server service account can write to. UNC paths must be reachable as the service account, not your interactive user.
  • Job fails immediately, no video. The export range may not overlap any recorded footage. This is logged but the job is still recorded as successful with 0 bytes.
  • AVI + Encrypt rejected at save. AVI export does not support encryption. Switch to XProtect, or disable encryption.
  • Progress bar stuck at 0%. Encryption + signing on very large exports can take a while before progress moves. Check the Milestone System Log for category Auto Exporter.
