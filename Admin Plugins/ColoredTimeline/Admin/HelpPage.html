Render any pair of camera events as colored ribbons on the Smart Client timeline
Open source - no warranty, use at own risk Per-camera selectable events

How It Works

XProtect's built-in timeline only shows motion sequences detected by the Recording Server's built-in motion detection. Edge motion (motion detected by the camera firmware) and other camera-fired events do not appear on the playback timeline.

This plugin closes that gap. Each Timeline Rule pairs a start event with a stop event for a specific list of cameras and a chosen color. Whenever the user pans or zooms the timeline in Smart Client, the plugin queries the event log for paired start / stop events on each camera and paints the resulting intervals as a ribbon in the configured color.

Prerequisites

Device-event retention

In Management Client open Tools > Options > Alarms and Events and set device-event retention to at least as many days as your video retention. Ribbons are rendered from historical event-log queries; events that have aged out will not appear even if the recording still exists.

Smart Client setting

In Smart Client open Settings > Timeline and set Additional data to Show. Without this, plugin-provided ribbons are silently suppressed.

Unblock files (manual install only)

When installing by copy instead of the MSI, Windows may flag DLLs from internet downloads. Right-click each DLL > Properties > Unblock, or install via the MSI to avoid this.

Getting Started

  1. Click on the Timeline Rules node in the tree on the left.
  2. Right-click and select Create New to add a rule.
  3. Give the rule a name and pick a ribbon color.
  4. Click Add Camera... and select the cameras this rule should apply to.
  5. Pick the Start event and Stop event from the dropdowns. The dropdowns are populated automatically from the trigger events of the cameras you added.
  6. Click Save. Smart Client picks up the change within a few seconds.

Camera Selection

Each rule applies only to the cameras listed under Cameras. A camera not in any rule's list gets no ribbon. The same camera can appear in multiple rules, in which case multiple ribbons are stacked on its timeline (one per matching rule).

  • Use Add Camera... to open the Milestone camera picker.
  • Use Remove to drop a camera from the list.
  • The Start / Stop event dropdowns refresh automatically from the trigger events exposed by the cameras you have selected.

Picking the Right Event

The Start event and Stop event dropdowns list every trigger event that the selected cameras expose, so for typical edge-motion setups you will find entries like Motion Started Driver / Motion Stopped Driver, or vendor-specific names for object analytics, line crossings, etc.

The dropdown is populated automatically when you add or remove cameras. It merges two sources: the configured trigger events on each camera, and the messages of events that have actually fired in the last 7 days (queried from the Event Server's event log). That second source is what surfaces driver-fired events like Motion Started Driver which otherwise have no configuration item.

Free-text is also accepted in the Start / Stop fields, in case an event hasn't been observed yet or has aged out of the event log.

How a Ribbon Is Drawn

When the user pans or zooms the Smart Client timeline, Smart Client asks every registered TimelineSequenceSource for sequences in the visible time window. For each rule that applies to the camera in that view item, the plugin runs two parallel event-log queries (start and stop) scoped to the camera and the configured message text, then pairs them into intervals.

  • Each start is paired with the next stop at or after its timestamp.
  • An open interval (start without a matching stop) is extended to the end of the visible window.
  • Overlapping starts (a second start before the previous stop) are skipped.
  • The interval is drawn as a colored ribbon in the configured Ribbon color.

Architecture

The plugin runs in two environments:

Management Client (Admin UI)

Provides the configuration interface you are using now. Lets you create, edit, and delete timeline rules. Each rule pairs an event start / stop with a list of cameras and a ribbon color, stored as a MIP item.

Smart Client (Background Plugin)

Runs invisibly in the Smart Client. For every camera view item that opens, it walks the configured rules and registers one TimelineSequenceSource per rule whose camera list includes that view item's camera. Each source listens for timeline range requests and answers with paired start / stop intervals queried from the event log.

Limitations

  • Historical events only. Live ribbon updates require a pan / zoom or seek to refresh the visible range.
  • One color per rule. To stack ribbons of different colors on the same camera, create multiple rules.
  • Events must be retained. See Prerequisites - device-event retention must cover the playback range you want to colorize.
  • Smart Client toggle required. "Additional data" must be set to Show in the timeline settings, otherwise plugin ribbons are not rendered.
