How It Works

The Auditor plugin monitors operator activity in the Smart Client and tracks playback mode changes, export operations, and independent playback on individual cameras.

Administrators configure audit rules in the Management Client to select which users are monitored. Each rule has two independent sets of toggles: Reason Prompts force the user to provide a reason (written to the Milestone audit log), and Event Triggers fire XProtect analytics events that can be used in rules and alarms.

Getting Started

  1. Click on the Audit Rules node in the tree on the left.
  2. Right-click and select Create New to add an audit rule.
  3. Give the rule a name, select the users to monitor, configure reason prompts and event triggers, then click Save.

Camera Selection

By default, audit rules apply to all cameras. To restrict a rule to specific cameras, enable the Specify Cameras checkbox and add cameras using the camera picker. When enabled, reason prompts and event triggers will only fire when the user interacts with one of the listed cameras.

  • Playback – triggered only if a specified camera is in the current view
  • Export – triggered only if a specified camera is in the current view
  • Independent Playback – triggered only for the specific camera

Reason Prompts (Audit Log)

When enabled, the user is shown a dialog and must enter a reason. The reason is written to the Milestone system audit log.

  • Playback Reason Prompt - When entering playback mode
  • Export Reason Prompt - When entering the export workspace
  • Independent Playback Reason Prompt - When enabling independent playback on a camera

Event Triggers (Rule Events)

When enabled, an XProtect analytics event is fired that can be used in rules, alarms, and notifications.

  • Playback Trigger Event - User switches to playback mode
  • Export Trigger Event - User enters the export workspace
  • Independent Playback Trigger Event - User enables independent playback on a camera

Architecture

The plugin consists of three components that run in different processes:

Management Client (Admin UI)

Provides the configuration interface you are using now. Lets you create, edit, and delete audit rules. Each rule defines which users are monitored, which reason prompts are shown, and which events are triggered.

Smart Client (Background Plugin)

Runs invisibly in the Smart Client. Monitors mode changes, export operations, and independent playback. Shows reason prompts when configured and sends audit reports to the Event Server via MessageCommunication.

Event Server (Background Plugin)

Receives audit reports from Smart Clients. Writes reason prompts to the Milestone audit log and fires XProtect analytics events when event triggers are enabled. These events can trigger rules, notifications, and alarms.
