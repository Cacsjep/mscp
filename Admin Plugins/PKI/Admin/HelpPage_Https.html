TLS certificates for the HTTPS endpoints exposed by XProtect™ servers
Validity max 397 days for browser trust

What lives here

HTTPS certificates are leaf certs used by browser-facing HTTPS endpoints - typically the Mobile Server's web client, the API Gateway, or any custom IIS site fronting Milestone APIs. They are signed by a Root or Intermediate CA from this vault.

For internal service-to-service TLS (Recording Server - Mgmt Server, etc.) prefer the Service Certificates folder instead.

Required fields

  • Sign with - an existing CA (Root or Intermediate) whose key still lives here.
  • Display name - the label in the tree and in certlm.msc.
  • Validity - modern browsers reject certs longer than 397 days. Pick 365 days for safety.
  • Subject Alternative Names - required. Add every hostname clients will use plus any direct-IP access. Click Add this machine's hostname and IPs to populate from the local machine.

Deployment

The PKI Cert Installer routes HTTPS certs to the local Windows Personal store and grants NETWORK SERVICE (or whichever account you pick) Full Control over the private-key file under %ProgramData%\Microsoft\Crypto\. After installing, open the Server Configurator on the target server to bind the cert to the relevant HTTPS endpoint.
