What lives here

Root certificates are the trust anchor for everything else in this vault. They are self-signed - their issuer is themselves - and they sign Intermediate CAs and (optionally) leaf certs directly.

Right-click Root Certificates in the tree and pick Add new... to issue one, or run Auto setup from the Overview pane to get one created automatically.

Required fields

  • Display name - the label shown in the tree, in Issuer dropdowns, and inside certlm.msc after the Cert Installer deploys it. Auto setup uses MSCP PKI (yyyy.MM) - Root CA.
  • Validity - recommended 10 to 20 years. The Root is expensive to rotate (every dependent cert needs to be re-issued), so don't pick too short.
  • Key algorithm - RSA-3072 or ECDSA-P256 are sound defaults. Keep the algorithm consistent with what your client devices (cameras, NVRs) accept for 802.1X if applicable.

Workflow

  1. Issue at most one or two Roots for the lifetime of the site.
  2. Create one or more Intermediate CAs signed by this Root and use the Intermediate to sign all leaf certs (HTTPS, Service, 802.1X). That way the Root key can stay offline-style and you only ever need to redistribute the Intermediate when renewing.
  3. Deploy the Root to the Windows Trusted Root Certification Authorities store on every machine that needs to validate the chain. The PKI Cert Installer EXE handles this automatically (Root certs are routed to the Root store).

Deletion rules

A Root cannot be deleted while any cert in the vault still names it (by thumbprint) as its issuer. Either delete those dependents first, or include them in the same multi-select delete on the Overview page.
