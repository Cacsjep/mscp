What lives here

This is a grouping node. It contains the three folders below, each holding leaf certificates signed by a Root or Intermediate from the CA Certificates branch.

Sub-folderUse it for
HTTPS CertificatesBrowser-facing TLS endpoints (Mobile Server, API Gateway, custom IIS sites). Validity capped at 397 days for browser trust.
802.1X CertificatesClient identity certs presented by cameras / NVRs / servers to a managed switch RADIUS for port authentication.
Service CertificatesInternal service-to-service TLS (Recording Server, Mgmt Server, Event Server, Mobile Server). Not bound by browser validity rules.

Generation pattern

  1. Drill into one of the folders above and right-click to add a new cert, or run Generate service certificates from the Overview pane to bulk-issue a batch.
  2. Pick an existing CA from the Sign with dropdown. Only CAs whose private key is still on this server are listed.
  3. Fill in Subject Alternative Names. The Add this machine's hostname and IPs button populates the SAN list from the local machine.
  4. Save. The PFX is generated and persisted to MIP item config.

Deployment

Run the PKI Cert Installer EXE on the target XProtect server, sign in to the management server, tick the certs you want, and click Install. The EXE routes each cert to the correct Windows store (Root certs to Root, Intermediates to CA, leaves to My) and grants the chosen service account Full Control on the private-key file.
