CAs signed by a Root, used to issue day-to-day leaf certs
Recommended issuer for leaves

What lives here

Intermediate CAs sit between the Root and the leaf certs. They are signed by an existing Root in the Root Certificates folder and can in turn sign HTTPS, Service, or 802.1X leaf certificates.

Using an Intermediate is the recommended pattern: if an Intermediate's key is ever compromised you can revoke just that one and re-issue, without having to replace the Root cert in every Windows trust store.

Required fields

  • Sign with - pick an existing Root CA whose private key still lives on this server. Roots imported as cert-only are not eligible.
  • Display name - e.g. MSCP PKI 2026 - Issuing CA.
  • Validity - commonly 5 to 10 years. Should expire well before the Root.
  • Key algorithm - usually matches the Root's.

Deployment

Intermediates are routed to the Windows Intermediate Certification Authorities store by the PKI Cert Installer. They are also bundled into the chain inside every PFX exported from this vault, so distributing the leaf cert with chain is enough for most clients to validate it.

Deletion rules

An Intermediate cannot be deleted while any leaf cert in the vault still names it as its issuer. Delete the leaves first or include them in a multi-select delete from the Overview page.
